# Live2D 模型配置文件规范

> 本文档定义了 `src/config/live2d/` 目录下模型配置文件的编写规则。
> 参考模型：[akari.ts](./akari.ts)（当前默认模型）、[universal.ts](./universal.ts)

---

## 目录结构

```
src/config/live2d/
├── 配置规则.md        # 本文档 — 配置文件规范
├── default.ts         # 默认模型入口，re-export 当前使用的模型
├── universal.ts       # 万能自定义模型配置
├── akari.ts           # akari 模型配置（当前默认）
├── moran.ts           # moran 模型配置
└── <新模型>.ts        # 新增模型在此添加
```

---

## 文件规则

### 1. 每个模型一个 `.ts` 文件

文件名使用英文小写 + 下划线（如 `akari.ts`、`my_model.ts`），
与 `public/live2d-models/` 下的模型目录名对应。

### 2. 文件结构模板

```typescript
/**
 * Live2D 模型配置 — <模型显示名>
 *
 * <简要描述：来源、纹理数、表情数、动作数等关键信息>
 * <特殊说明：如 model3.json 未定义哪些字段、需要显式列出的原因>
 *
 * 特性：
 * - <关键配置项 1>：<解释为什么这样配置>
 * - <关键配置项 2>：<解释为什么这样配置>
 */
import type { ModelInfo } from "../../types/live2d";

const config: ModelInfo = {
  // ── 必填 ──
  name: "<模型显示名>",
  url: "live2d-models/<目录>/<文件名>.model3.json",

  // ── 视觉调整 ──
  kScale: 0.85, // 缩放因子，默认 1.0
  initialXshift: 0, // 水平偏移（像素）
  initialYshift: 0, // 垂直偏移（像素）
  fitMode: "contain", // "contain" | "cover"，默认 "contain"

  // ── 动画 ──
  // idleMotionGroupName: "Idle",  // 空闲动作组（默认 "Idle"）
  // defaultEmotion: "happy",      // 默认表情（名称或索引）

  // ── 交互 ──
  pointerInteractive: true,
  tapMotions: {
    // Body: "<动作组名>",  // 点击身体区域触发
    // Head: "<动作组名>",  // 点击头部区域触发
  },

  // ── 资源清单（当 model3.json 未定义时使用） ──
  expressionFiles: [
    // "expressions/<文件名>.exp3.json",
  ],
  motionFiles: [
    // { group: "<动作组>", file: "motions/<文件名>.motion3.json" },
  ],
};

export default config;
```

### 3. 必填字段

| 字段   | 类型     | 说明                                      |
| ------ | -------- | ----------------------------------------- |
| `name` | `string` | 模型显示名称，用于 UI 展示                |
| `url`  | `string` | model3.json 的路径，相对于 `public/` 目录 |

### 4. 视觉调整字段

| 字段            | 类型                   | 默认值      | 说明                                                   |
| --------------- | ---------------------- | ----------- | ------------------------------------------------------ |
| `kScale`        | `number`               | `1.0`       | 缩放因子。模型过大时减小（如 `0.85`），过小时增大      |
| `initialXshift` | `number`               | `0`         | 水平偏移（像素），正数向右                             |
| `initialYshift` | `number`               | `0`         | 垂直偏移（像素），正数向下                             |
| `fitMode`       | `"contain" \| "cover"` | `"contain"` | 适配模式。`contain`=完整可见，`cover`=填满容器可能裁剪 |

### 5. 动画字段

| 字段                  | 类型               | 默认值   | 说明                                                                                       |
| --------------------- | ------------------ | -------- | ------------------------------------------------------------------------------------------ |
| `idleMotionGroupName` | `string`           | `"Idle"` | 空闲动作组名。设为 `""` 可禁用空闲动画，由 CubismBreath + CubismEyeBlink 提供默认呼吸/眨眼 |
| `defaultEmotion`      | `string \| number` | 无       | 启动时自动应用的表情名称或索引                                                             |

### 6. 交互字段

| 字段                 | 类型                     | 默认值  | 说明                                                       |
| -------------------- | ------------------------ | ------- | ---------------------------------------------------------- |
| `pointerInteractive` | `boolean`                | `true`  | 是否启用鼠标/触摸交互                                      |
| `tapMotions`         | `Record<string, string>` | `{}`    | 点击区域→动作映射。如 `{ Body: "嗷呜", Head: "点头数数" }` |

> **注**：长按拖动模型 / 悬停滚轮缩放不在模型配置中，是全局交互开关，
> 由 `live2dStore` 的 `modelDragEnabled` / `wheelZoomEnabled` 控制（localStorage 持久化，默认开启）。
> `scrollToResize` 字段已废弃，滚轮缩放统一走 store 开关。

### 7. 资源清单字段

当 `model3.json` 未定义 `Expressions` 或 `Motions` 字段时（常见于 VTube Studio 导出的模型），
需要通过以下字段显式列出：

| 字段              | 类型                                | 说明                                                                  |
| ----------------- | ----------------------------------- | --------------------------------------------------------------------- |
| `expressionFiles` | `string[]`                          | 表情文件路径列表，相对于模型目录。如 `"expressions/星星眼.exp3.json"` |
| `motionFiles`     | `{ group: string; file: string }[]` | 动作文件列表。`group` 是动作组名，`file` 是相对于模型目录的路径       |

> **判断是否需要显式列出**：检查 `model3.json` 中是否有 `FileReferences.Expressions` 或 `FileReferences.Motions` 字段。
> 如果没有，或者字段值为空数组，则需要通过配置文件显式指定。

### 8. 注释规范

- **文件头注释**：描述模型来源、纹理/表情/动作数量、特殊说明
- **特性注释**：列出关键配置项及其原因（如 `fitMode="cover"` 的原因）
- **字段注释**：非显而易见的字段用行内注释解释
- **统一注释用语**：
  - `model3.json 未定义 Expressions 字段，显式列出`
  - `model3.json 未定义 Motions 字段，显式列出`
  - `不绑定空闲动作，由 CubismBreath + CubismEyeBlink 提供默认呼吸/眨眼`

### 9. 字段排列顺序

各字段按以下顺序排列（提高可读性）：

```
name
url
kScale / initialXshift / initialYshift / fitMode     ← 视觉调整
idleMotionGroupName / defaultEmotion                   ← 动画
pointerInteractive / tapMotions                        ← 交互
expressionFiles                                        ← 资源清单 - 表情
motionFiles                                            ← 资源清单 - 动作
```

## 如何切换默认模型

编辑 `default.ts`，修改 import 源即可：

```typescript
// 当前使用 akari（纯英文文件名，兼容 Android APK 打包编码）
export { default } from "./akari";

// 切换到 universal（万能自定义模型）：
// export { default } from "./universal";

// 切换到 moran：
// export { default } from "./moran";
```

所有组件通过 `import { DEFAULT_MODEL_INFO } from "../../config/live2d"` 引用，
切换默认模型无需修改任何组件代码。

## 如何添加新模型

1. 将模型文件放入 `public/live2d-models/<模型目录>/`
2. 在 `src/config/live2d/` 下创建 `<模型名>.ts`
3. 按照本文档的模板编写配置
4. 如果要设为默认，修改 `default.ts` 的 import 源

### 检查清单

- [ ] `model3.json` 路径是否正确（相对于 `public/`）
- [ ] 纹理文件是否存在（检查 `model3.json` 中的 `FileReferences.Textures`）
- [ ] 是否需要显式列出 `expressionFiles`（检查 `model3.json` 的 `FileReferences.Expressions`）
- [ ] 是否需要显式列出 `motionFiles`（检查 `model3.json` 的 `FileReferences.Motions`）
- [ ] `kScale` 是否需要调整（在浏览器中预览后确定）
- [ ] 点击交互是否配置（`tapMotions`）
